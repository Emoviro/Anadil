# V0.2 Merge Notes

## Ozet

V0.2, Anadil'in native compiler hattina dinamik `metin` altyapisini ve RC
temelini ekler. V0.1'deki native executable uretme hattinin ustune
length-prefixed `metin` layout'u, heap allocation ile `metin + metin`,
`uzunluk(metin) -> sayi` builtin'i ve kapsam/arguman/return cleanup akislari
eklendi.

## Baslica Degisiklikler

- Dynamic `metin + metin` runtime heap allocation ile calisir.
- Static ve heap `metin` degerleri length-prefixed runtime layout'una tasindi.
- Runtime RC primitive'leri kullanilir: `tahsis`, `paylas`, `birak`.
- Native backend function scope, branch scope, loop scope, return, function
  arg, builtin arg, unused expression ve nested concat cleanup akislari icin
  RC emit eder.
- `uzunluk(metin) -> sayi` interpreter, IR ve native backend'de desteklenir.
- `anadil ir` runtime metin operasyonlarini acik isimlerle gosterir.
- `examples/metin_v02.ana` V0.2 metin demosu olarak eklendi.
- RC audit checklist'i `Docs/native_compiler.md` icinde belgelendi.

## Bilinen Sinirlar

- `uzunluk(metin)` V0.2 MVP'de byte length dondurur; Unicode grapheme/karakter
  sayimi sonraki metin API katmanina kalir.
- Last-use optimizasyonu V0.2.1'e ertelendi.
- Dizi/yapi MVP kapsami V0.3 veya ayri faz olarak tutulacak.
- Cycle handling yoktur; mevcut `metin` tipi cycle uretmedigi icin V0.2
  kapsaminda sorun olusturmaz.
